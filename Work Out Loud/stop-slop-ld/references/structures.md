# Structures to Avoid: Learning Design Edition

## The Composite Persona Setup

Introducing a fictional character to stand in for the learner.

| Pattern | Problem |
|---------|---------|
| "Meet Alex, a project manager at a mid-sized company..." | Alex is nobody. Put the learner in the seat directly. |
| "Consider Sarah, who faces a common challenge..." | Same. The distance drains stakes. |
| "In our scenario, Jordan must decide..." | "You must decide" is harder and better. |
| "Think about a typical employee in this situation..." | There's no typical employee. Use second person. |

**Instead:** "You're three days from the project deadline and the client just changed scope." The reader is already there.

---

## The Importance Announcement

Declaring that something matters without showing why.

| Pattern | Problem |
|---------|---------|
| "This skill is critical in today's workplace" | Every skill gets this line. It means nothing. |
| "Understanding X has never been more important" | Always the same claim. Always vague. |
| "In a rapidly changing world, X matters more than ever" | Filler. What changes? For whom? When? |
| "This has significant implications for..." | Name the implications. |
| "The stakes are high when it comes to X" | Name the stakes. |

**Instead:** Name the specific consequence. "Skip this step and the audit flags the entire record." That's a stake.

---

## Binary Learning Contrasts

Creating false drama between what learning is "not" versus what it "really is."

| Pattern | Problem |
|---------|---------|
| "This isn't a course about X. It's about Y." | Telegraphed reversal. Just say Y. |
| "Training doesn't change behavior. Practice does." | State the point directly. |
| "It's not about knowing the rules. It's about applying them." | Combine into one direct sentence. |
| "This isn't just information — it's a framework." | If it's a framework, say what it does. |

**Instead:** State Y directly. Drop the negation setup.

---

## Rhetorical Question Openers

Asking questions you're about to answer anyway.

| Pattern | Problem |
|---------|---------|
| "Have you ever wondered why...?" | Either they have or they haven't. Get to the answer. |
| "What would you do if...?" | Use a scenario with actual choices, not a rhetorical opener. |
| "Why does this matter?" | Answer it without asking it first. |
| "What does good X actually look like?" | Show it. Don't announce you're about to show it. |
| "How do we know when we've succeeded?" | Same — state the criteria. |

**Instead:** Make the point. Let learners draw conclusions.

---

## The Learning Objective Parade

Opening a module by listing everything that will be covered.

| Pattern | Problem |
|---------|---------|
| "In this course, you will: [bullet] [bullet] [bullet]" | This is navigation, not instruction. It front-loads anxiety, not readiness. |
| "After completing this module, learners will be able to: [list of 8 things]" | Eight objectives means no objective. |
| Objectives as a warm-up ritual with no return | If you list them and never reference them again, cut them. |

**Instead:** State the one thing this content makes possible. If there are multiple real outcomes, scope the module.

## The Sandwich Feedback Structure

In scenario-based learning, wrapping every incorrect answer in encouragement.

| Pattern | Problem |
|---------|---------|
| "Not quite! But good thinking. Here's what to consider..." | The softening undercuts the correction. |
| "That's a reasonable choice, but in this situation..." | If it's wrong, say why it's wrong. |
| "You might choose this in some situations, but here..." | Hedging the feedback trains learners to hedge their decisions. |

**Instead:** Tell them what happens. "That response escalates the conflict. Here's why..." Consequences teach. Encouragement doesn't.

## False Agency in Content

Giving abstract concepts or systems human verbs.

| Pattern | Problem |
|---------|---------|
| "The data drives the decision" | Data doesn't drive anything. The analyst decides. |
| "The framework guides practitioners through..." | Frameworks don't guide. You use them to guide yourself. |
| "The process ensures quality" | Processes don't ensure anything. People following them do. |
| "The policy prevents mistakes" | Policies don't prevent. People applying them do — or don't. |
| "This module will equip you with..." | Modules don't equip. You practice until you can do something. |
| "The course takes you through..." | Courses don't take. You work through them. |

**Instead:** Name the human. "When the analyst reads the trend, they decide whether to escalate." Specific agents, specific actions.

---

## Narrator-from-a-Distance

Floating above the content instead of putting the reader in it.

| Pattern | Problem |
|---------|---------|
| "Many employees struggle with this" | Who? Put the reader there instead. |
| "Organizations often fail to..." | Which ones? Why? |
| "This is a common challenge in the workplace" | Name the challenge directly. |
| "People tend to underestimate..." | "You probably underestimate..." is more honest and direct. |
| "Research shows that..." | Cite it or cut it. Vague appeals to research are throat-clearing. |

**Instead:** Put the reader in the room. "You've just received the complaint. You have four hours before it escalates to legal." That's a scenario. "Many employees face challenges with complaint handling" is not.

---

## The Three-Part Structural Template

Most AI-written instructional content follows one rhythm endlessly.

| Pattern | Problem |
|---------|---------|
| Topic sentence → two supporting bullets → summary sentence | Predictable. Readers stop reading. |
| Three examples every time | Two is usually enough. One done well beats three done averagely. |
| Every section the same length | Importance isn't uniform. Let length reflect weight. |
| Every slide: title + 4 bullets | If everything fits in bullets, you're not teaching — you're listing. |

**Instead:** Let the content shape the structure. Short where simple. Long where complex. Visual where spatial. No formula.

---

## Passive Learning Objectives

Objectives that describe the course's activity rather than the learner's performance.

| Pattern | Problem |
|---------|---------|
| "Learners will be introduced to..." | That's what the course does. What does the learner do? |
| "Participants will have the opportunity to practice..." | Opportunity isn't an outcome. |
| "This module covers..." | That's a syllabus entry, not an objective. |
| "By the end, you'll have a better understanding of..." | "Better" is unmeasurable. Name the performance. |

**Instead:** "Given a sample dataset with anomalies, identify and flag the three most likely error types." That's an objective — observable, specific, tied to a context.
